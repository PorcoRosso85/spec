# リポジトリ構造と責務の正 SSOT

このファイル docs/structure/index.md は 現在の正 のツリーと責務マッピングを示す場所
ここがずれていたらそれは drift とみなす

このリポジトリは 1人と AI 体制で壊さず拡張するために 正の情報源 SSOT を三つに分ける

- catalog 使える責務スロットの全集合 カタログ
- adr どのスロットをいつどう採用 移譲 破棄するか その理由のログ
- skeleton 今回の形 現行の構成スナップショットと契約

src は skeleton の指示どおりにだけ触ってよい
この順番 catalog → adr → skeleton → src を踏まずにコードを増やすのは禁止

この三本立てと CI の境界 そして PR に求める手順の詳細は
[1人と AI 体制で壊さず拡張し続けるための構造ルールと CI 境界](structure-1人AI体制で壊さず拡張し続ける.md) を参照
このルールに従わない変更は受け付けない前提でレビューする

簡単なディレクトリサマリ

```text
catalog/   事業カタログ 責務スロット全集合
adr/       意思決定ログ なぜそう置くか
skeleton/  今回の形と spec
src/       実装本体 skeleton で許された場所だけ
```

index.md 自体は 常に最新の正 を示す義務を持つ
このファイルが古い状態は drift であり CI で検知対象になる予定
